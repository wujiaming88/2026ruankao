## 一、大模型/LLM（⭐⭐⭐⭐⭐ 必考 3-5题）

### 1.1 Transformer架构

| 项目 | 内容 |
|------|------|
| **一句话** | 基于自注意力机制的深度学习架构，所有主流大模型的基础 |
| **核心组件** | 编码器(Encoder) + 解码器(Decoder)，多头自注意力(Multi-Head Self-Attention)，前馈神经网络(FFN)，位置编码(Positional Encoding) |
| **关键特征** | 并行计算（不像RNN串行）、长距离依赖建模、可扩展性强 |
| **vs RNN** | RNN串行处理序列→慢；Transformer并行处理→快，且能捕获更长距离关系 |
| **vs CNN** | CNN擅长局部特征（图像）；Transformer擅长全局关系（文本/多模态） |

```
考题角度：
- 大模型的底层架构是什么？→ Transformer
- Transformer的核心机制是什么？→ 自注意力(Self-Attention)
- Transformer相比RNN的优势？→ 并行计算、长距离依赖
```

### 1.2 大模型分类

| 类型 | 代表 | 特点 |
|------|------|------|
| **自回归模型** | GPT系列 | 从左到右生成，擅长文本生成 |
| **掩码语言模型** | BERT | 双向理解，擅长理解/分类任务 |
| **编码-解码模型** | T5、BART | 输入理解+输出生成，擅长翻译/摘要 |
| **多模态模型** | GPT-4V、Gemini | 同时处理文本+图像+音频 |
| **混合专家(MoE)** | Mixtral、DeepSeek | 多个专家网络，按需激活，高效推理 |

```
考题角度：
- GPT属于什么类型？→ 自回归(Autoregressive)
- BERT和GPT的区别？→ BERT双向理解，GPT单向生成
- MoE的核心思想？→ 多专家按需激活，降低计算成本
```

### 1.3 大模型关键技术

| 技术 | 说明 |
|------|------|
| **预训练(Pre-training)** | 在大规模无标注数据上学习通用知识 |
| **微调(Fine-tuning)** | 在特定任务数据上调整模型参数 |
| **RLHF** | 基于人类反馈的强化学习，让模型输出更符合人类偏好 |
| **Prompt Engineering** | 通过设计提示词引导模型输出，不修改参数 |
| **上下文学习(ICL)** | 在提示中给几个示例，模型就能学会新任务 |
| **思维链(CoT)** | 让模型分步推理，提高复杂问题准确率 |

```
考题角度：
- 不修改模型参数的技术？→ Prompt Engineering / ICL
- 让大模型对齐人类价值观的技术？→ RLHF
- 大模型的训练流程？→ 预训练→微调→对齐(RLHF)
```

### 1.4 大模型工程化

| 技术 | 说明 |
|------|------|
| **模型量化(Quantization)** | 降低参数精度(FP32→INT8)，减小模型体积，加速推理 |
| **模型蒸馏(Distillation)** | 大模型(教师)训练小模型(学生)，知识迁移 |
| **模型剪枝(Pruning)** | 去除不重要的参数/连接，压缩模型 |
| **稀疏编码** | 只激活部分神经元，降低计算量 |

```
口诀："量蒸剪稀"（量化、蒸馏、剪枝、稀疏）= 模型压缩四件套
考题角度：2025上半年已考端侧AI + 模型压缩，2026年大概率再考
```

## 二、RAG 检索增强生成（⭐⭐⭐⭐⭐ 必考 1-2题）

| 项目 | 内容 |
|------|------|
| **一句话** | 先检索相关知识，再让大模型基于检索结果生成回答 |
| **解决的问题** | 大模型的"幻觉"(编造事实)、知识过时、缺乏专业领域知识 |
| **核心流程** | 用户提问 → 向量检索(Embedding+向量数据库) → 取回相关文档 → 拼入Prompt → LLM生成回答 |
| **关键组件** | Embedding模型(文本→向量)、向量数据库(Milvus/Pinecone/FAISS)、LLM |
| **vs 微调** | 微调=改模型参数（贵、慢）；RAG=给模型喂资料（便宜、实时更新） |

```
RAG流程图：
用户问题 → Embedding → 向量检索 → 取回Top-K文档 → 构建Prompt → LLM生成 → 回答

考题角度：
- RAG的核心目的？→ 减少幻觉，引入外部知识
- RAG和微调的区别？→ RAG不改参数，实时更新；微调改参数，成本高
- 向量数据库的作用？→ 存储和检索文本向量(Embedding)
```

## 三、AI Agent 智能体（⭐⭐⭐⭐⭐ 必考 1-2题）

| 项目 | 内容 |
|------|------|
| **一句话** | 基于大模型的自主智能体，能感知环境、规划任务、调用工具、自主执行 |
| **核心能力** | 感知(Perception) → 规划(Planning) → 行动(Action) → 反思(Reflection) |
| **关键组件** | LLM(大脑) + 记忆(Memory) + 工具(Tools) + 规划(Planning) |
| **记忆类型** | 短期记忆(对话上下文) + 长期记忆(向量数据库存储) |
| **多Agent协作** | 多个Agent分工协作，如"产品经理Agent+开发Agent+测试Agent" |

```
Agent架构：
感知环境 → LLM推理+规划 → 选择工具 → 执行动作 → 观察结果 → 循环

vs 传统软件：传统=人写规则执行；Agent=AI自主决策执行
vs RAG：RAG只检索+生成；Agent能规划+调用工具+执行多步任务

考题角度：
- AI Agent的核心组件？→ LLM+记忆+工具+规划
- Agent和传统软件的区别？→ 自主决策 vs 预定义规则
```

## 四、数据安全与隐私（⭐⭐⭐⭐ 大概率考 1-2题）

### 4.1 法律法规

| 法律 | 施行时间 | 核心内容 |
|------|---------|---------|
| **网络安全法** | 2017.6 | 网络运行安全、等级保护、关键信息基础设施保护 |
| **数据安全法** | 2021.9 | 数据分类分级、数据安全审查、重要数据出境评估 |
| **个人信息保护法** | 2021.11 | 个人信息处理规则、知情同意、最小必要原则 |
| **生成式AI管理办法** | 2023.8 | AI生成内容标识、训练数据合规、算法备案 |

```
口诀："网数个AI"（网络安全法→数据安全法→个人信息保护法→AI管理办法）
考题角度：
- 数据出境需要什么？→ 安全评估
- 个人信息处理的基本原则？→ 知情同意、最小必要
- AI生成内容需要什么？→ 标识标注
```

### 4.2 隐私计算技术

| 技术 | 说明 | 特点 |
|------|------|------|
| **联邦学习** | 数据不出本地，只交换模型参数/梯度 | "数据不动模型动" |
| **安全多方计算(MPC)** | 多方联合计算，各方看不到其他方数据 | 密码学保障 |
| **可信执行环境(TEE)** | 在CPU硬件级别隔离的安全区域执行 | 硬件保障（如Intel SGX） |
| **差分隐私** | 在数据中加噪声，保护个体隐私 | 统计层面保护 |
| **同态加密** | 对加密数据直接计算，无需解密 | 理论最强，但性能差 |

```
口诀："联安可差同"
考题角度：
- "数据不动模型动"是什么技术？→ 联邦学习
- 硬件级安全计算？→ 可信执行环境(TEE)
- 对加密数据直接运算？→ 同态加密
```

## 七、云原生进阶（⭐⭐⭐⭐ 大概率考 1-2题）

### 7.1 Serverless/函数计算

| 项目 | 内容 |
|------|------|
| **一句话** | 无需管理服务器，按函数执行次数/时长计费 |
| **特点** | 自动扩缩容、按需付费、事件驱动、无状态 |
| **代表** | AWS Lambda、阿里云函数计算、腾讯云SCF |
| **vs 容器** | 容器=你管应用，平台管OS；Serverless=你只管代码，平台管一切 |

### 7.2 服务网格(Service Mesh)

| 项目 | 内容 |
|------|------|
| **一句话** | 微服务之间通信的基础设施层，处理服务间的网络通信 |
| **核心模式** | Sidecar代理模式（每个服务旁挂一个代理） |
| **代表** | Istio + Envoy |
| **功能** | 服务发现、负载均衡、熔断、限流、链路追踪、mTLS加密 |
| **vs API网关** | API网关=南北流量(外→内)；服务网格=东西流量(内↔内) |

### 7.3 DevSecOps

| 项目 | 内容 |
|------|------|
| **一句话** | 在DevOps流程中内嵌安全，安全左移(Shift Left) |
| **vs DevOps** | DevOps=开发+运维；DevSecOps=开发+安全+运维 |
| **核心理念** | 安全不是最后检查，而是贯穿开发全流程 |
| **实践** | SAST(静态代码扫描)、DAST(动态测试)、SCA(依赖检查)、容器镜像扫描 |

```
考题角度：
- Serverless的计费方式？→ 按执行次数/时长
- Service Mesh的核心模式？→ Sidecar代理
- DevSecOps的核心理念？→ 安全左移
```

## 八、区块链进阶（⭐⭐⭐ 可能考 1题）

### 区块链六层架构（2025上半年案例已考）

```
1. 数据层 — 区块+链+哈希+Merkle树+时间戳+非对称加密
2. 网络层 — P2P网络+传播机制+验证机制
3. 共识层 — PoW/PoS/DPoS/PBFT
4. 激励层 — 挖矿奖励+交易手续费
5. 合约层 — 智能合约（自动执行）
6. 应用层 — DApp（金融/溯源/身份）

口诀："数网共激合应"
```

### 共识算法对比

| 算法 | 特点 | 适用 |
|------|------|------|
| **PoW** | 拼算力(挖矿)，安全但慢 | 公链（比特币） |
| **PoS** | 拼持币量，节能 | 公链（以太坊2.0） |
| **DPoS** | 代表投票，快 | 联盟链 |
| **PBFT** | 投票共识，确定性 | 联盟链/私有链 |

### 9.1 信息物理系统 CPS（⭐⭐⭐⭐ 教材新增Ch11.1，已考过）

```
考题角度：
- CPS的核心是什么？→ 3C融合（计算+通信+控制）
- CPS和IoT的区别？→ CPS强调闭环控制，IoT强调连接
- CPS的应用场景？→ 智能制造、自动驾驶

### 9.4 物联网 IoT + 5G（⭐⭐⭐ 十四五重点+教材Ch13.6）

| 项目 | 内容 |
|------|------|
| **IoT三层架构** | 感知层(传感器/RFID) → 网络层(通信传输) → 应用层(业务处理) |

```
口诀：IoT三层"感网应"，5G三场景"增超海"
考题角度：
- IoT的三层架构？→ 感知层、网络层、应用层
- 5G的URLLC场景适合什么？→ 自动驾驶、远程手术（超低延迟）
