## ⚠️ 设计模式易混对（必背判断口诀）

### 策略 vs 状态
```
主动选哪种方式 → 策略（选交通工具）
处于某状态行为自动变 → 状态（红绿灯）
```

### 适配器 vs 桥接
```
接口不兼容/第三方接入 → 适配器（转接头，事后补救）
两个维度独立变化 → 桥接（遥控器×电视，事前设计）
```

### 建造者 vs 模板方法
```
分步造一个复杂对象 → 建造者（组装电脑）
定义算法骨架子类重写步骤 → 模板方法（做菜食谱）
终极判断：得到东西→建造者；执行过程→模板方法
```

### 命令 vs 备忘录
```
存动作/操作为对象 → 命令（录像倒放）
存状态快照 → 备忘录（游戏存档）
终极口诀：命令存"动作"，备忘录存"状态"
```

### 外观 vs 代理 vs 中介者
```
多个子系统统一简化入口（外→内） → 外观（酒店前台）
控制单个对象访问（一→一） → 代理（保镖）
多个模块相互协调（内↔内） → 中介者（群聊管理员）
```

### 享元 vs 单例
```
全局只有1个实例 → 单例
按类型共享少量几个实例（节省内存） → 享元
```

### 工厂方法 vs 抽象工厂
```
创建一种产品 → 工厂方法
创建一族/一套产品 → 抽象工厂
```

## 🔴 网络OSI层次（4/8~4/9新增，硬伤区！）

### OSI七层 + 设备 + 协议 + 数据单元
```
7 应用层    | 网关      | HTTP/FTP/SMTP/DNS        | 数据/报文
6 表示层    | —        | 格式转换/加密/压缩           | 数据
5 会话层    | —        | 建立/管理/终止会话           | 数据
4 传输层    | —        | TCP/UDP                  | 段(Segment)
3 网络层    | 路由器    | IP/ICMP/ARP/OSPF         | 包(Packet)
2 数据链路层 | 交换机    | PPP/HDLC                 | 帧(Frame)
1 物理层    | 集线器    | —                        | 比特(Bit)
```

### 各层寻址方式
```
链路层 → MAC地址
网络层 → IP地址
传输层 → 端口号
```

### 地址解析四协议
```
ARP  = IP → MAC（地址解析）
RARP = MAC → IP（反向解析）
DNS  = 域名 → IP
DHCP = 动态分配IP
```

### 防火墙三类型对应层
```
包过滤     → 网络层（第3层）
状态检测   → 传输层（第4层）
应用代理   → 应用层（第7层）
口诀："包网状传应应"
```

### 攻击方式辨析
```
Spoofing = 伪造身份/IP，冒充（假快递员）
MITM     = 中间窃听+篡改（偷拆信件）
DoS      = 大量请求瘫痪（堵门）
Brute Force = 穷举密码（挨个试钥匙）
```

### 子网计算公式
```
可用主机数 = 2^主机位 - 2（减网络地址+广播地址）
```

### VLAN
```
VLAN = 逻辑隔离广播域（缩小，不是增大！）
同VLAN = 二层通信
跨VLAN = 必须经过三层（路由器/三层交换机）
```

### TCP vs UDP
```
TCP：可靠、有序、慢（网页、邮件、文件）
UDP：不可靠、无序、快（视频、游戏、DNS）
```

### SSL/TLS vs SSH vs IPSec
```
SSL/TLS → Web安全（HTTPS = HTTP + SSL/TLS）
SSH     → 远程登录安全
IPSec   → 网络层VPN加密
```

## 🔴 法律法规（4/10新增，薄弱区！）

### 著作权保护期限
```
自然人著作权：作者终生 + 死后50年
法人著作权：发表后50年
口诀："自然人生+50，法人发+50"
```

### 专利权保护期限
```
发明专利：申请日起20年
实用新型：申请日起10年
外观设计：申请日起15年
口诀："发20、实10、外15"
```

### 商标权
```
注册日起10年，到期可申请续展，每次续展10年
可无限续展
```

### 国家秘密保密期限
```
绝密级：不超过30年
机密级：不超过20年
秘密级：不超过10年
口诀："绝30、机20、密10"
```

### 软件著作权归属
```
职务开发（工作时间/公司资源）→ 归公司
委托开发（有合同）→ 按合同
委托开发（无合同）→ 归开发方
合作开发 → 共同所有
```

### 开源许可证
```
GPL = 最严格，传染性（衍生品必须也开源）
LGPL = 弱传染（库可被闭源调用）
BSD = 宽松，保留声明即可
Apache = 宽松，可商用，注明修改

口诀：GPL最严传染，MIT最松随便
```

## 🔴 数学算法（4/10新增，最大弱项！）

### IP地址分类
```
A类：8位网络 + 24位主机，255.0.0.0，首字节1-126
B类：16位网络 + 16位主机，255.255.0.0，首字节128-191
C类：24位网络 + 8位主机，255.255.255.0，首字节192-223
口诀："A8、B16、C24"（网络位从小到大）
```

### 可靠性计算
```
串联（都要工作）：R = R1 × R2 × R3（越串越低）
并联（一个就行）：R = 1 - (1-R1)(1-R2)(1-R3)（越并越高）
混合：先算并联部分，再与其他串联
```

### 二叉树公式
```
n₀ = n₂ + 1（叶子数 = 度为2的节点数 + 1）
度为1的节点数是干扰项！
完全二叉树叶子数：N偶数→N/2，N奇数→(N+1)/2
```

### 排序算法
```
不稳定排序："快选希堆"（快速、选择、希尔、堆）
O(nlogn)：快速（不稳定）、归并（稳定）、堆（不稳定）
O(n²)：冒泡（稳定）、插入（稳定）、选择（不稳定）
```

### 后缀表达式计算
```
遇数字 → 入栈
遇运算符 → 弹两个，先弹的在右，后弹的在左
结果入栈，最终栈里剩一个就是答案
```

### 容斥原理
```
两个集合：|A∪B| = |A| + |B| - |A∩B|
三个集合：|A∪B∪C| = |A|+|B|+|C| - |A∩B|-|A∩C|-|B∩C| + |A∩B∩C|
口诀："加减加"（单加、双减、三加）
```

### 关键路径
```
1. 列出所有从起点到终点的路径
2. 算每条路径的总工期
3. 最长的那条 = 关键路径
工期优化：压最便宜的任务，压到增加费用≥间接费用就停
```

### 死锁避免公式
```
最少资源 = 进程数 × (每进程需求-1) + 1
例：5进程各需2资源 → 5×(2-1)+1 = 6
```

## 🔴 软件工程补充（4/10新增）

### CMMI五级（阶段式）
```
1. 初始级（Initial）
2. 已管理级（Managed）
3. 已定义级（Defined）
4. 定量管理级（Quantitatively Managed）
5. 优化级（Optimizing）
口诀："初管定量优"
```

### WPDRRC安全模型
```
W — Warning    预警
P — Protect    保护
D — Detect     检测
R — Response   响应
R — Recovery   恢复
C — Counter    反击
口诀："预保检响恢反"
```

### OSI安全五服务
```
鉴别（Authentication）
访问控制（Access Control）
数据机密性（Confidentiality）
数据完整性（Integrity）
不可抵赖性（Non-repudiation）
口诀："鉴访机完抗"
注意：用"鉴别"不是"认证"
```

### 边缘计算特点
```
✅ 低延迟（本地处理）
✅ 降低功耗（减少传输）
✅ 提高安全（数据不上云）
✅ 减少带宽需求（不是提高带宽！）
```

### 管道过滤器风格
```
✅ 优点：支持并行执行、可复用、易维护
❌ 脆弱性：不适合交互式应用、格式转换开销大、难共享状态
注意："不能并行"是错误说法！
```

### 信息安全五属性
```
保密性：不被偷看（加密）
完整性：不被篡改（哈希/签名）
可用性：需要时能用（冗余/备份）
不可否认性：做了不能赖（数字签名）
可控性：可管可控（访问控制）
```

### 认证三因素
```
你知道的：密码、PIN码
你拥有的：手机、USB Key、令牌
你是谁：指纹、虹膜、人脸
双因素 = 必须来自不同类别！
```

### 网络拓扑
```
星型：中心节点连所有，中心挂全挂
环型：首尾相连成环，一断全断
总线型：共享一条线，线断分割
网状型：多路径互连，可靠性最高
```

### IPv4 vs IPv6
```
IPv4：32位，约43亿地址
IPv6：128位，约3.4×10^38地址
IPv6 = IPv4的2^96倍（不是4倍！）
IPv6取消首部校验和，内置IPSec
```

### OSI七层 vs TCP/IP四层
```
应用+表示+会话 → 应用层
传输层 → 传输层
网络层 → 网际层
数据链路+物理 → 网络接口层
```

### 数字信封
```
非对称加密传递对称密钥（安全）
对称加密加密实际数据（高效）
组合 = 既安全又高效
```

### 数字签名
```
发送方：私钥签名
接收方：发送方公钥验证
```

## 🔴 大数据+AI+区块链（4/8新增）

### 数据仓库 vs 数据湖
```
仓库 = 写时模式（Schema on Write）→ 清洗后 → 质量高
数据湖 = 读时模式（Schema on Read）→ 原始数据 → 质量低
```

### 四大神经网络
```
CNN = 图像（卷积、空间特征）
RNN = 序列（文本、语音、时间序列）
GAN = 生成（生成图片、换脸、对抗）
Transformer = 大模型（GPT、BERT、注意力机制）
```

### 大数据计算框架延迟排序
```
MapReduce：分钟~小时（批处理）
Hive：分钟级（SQL查询，底层MR）
Spark：秒级（内存微批）
Flink：毫秒级（真正流处理）← 毫秒选Flink！
```

### Hadoop三核心
```
HDFS = 存数据（分布式文件系统）
YARN = 管资源（调度框架）
MapReduce = 算数据（计算框架）
```

### Lambda vs Kappa
```
Lambda = 批处理 + 流处理两条路并行
Kappa  = 只用流处理一条路
```

### 区块链四核心
```
共识机制：PoW(拼算力,慢) / PoS(拼持币,快) / PBFT(投票,联盟链)
智能合约：自动执行的合约代码
哈希算法：保证不可篡改
数字签名：验证交易者身份
```

### 微服务事务
```
微服务场景 → Saga模式（最终一致性、补偿机制）
分布式数据库 → 2PC两阶段提交（强一致性、全局锁）
```
